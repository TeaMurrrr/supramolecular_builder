# Supramolecular Pipeline

Автоматизированный конвейер для моделирования, эмпирического докинга и квантово-химической оптимизации супрамолекулярных комплексов типа «Хозяин-Гость» (Host-Guest), а также изолированных молекулярных систем.

Конвейер объединяет топологический анализ RDKit, эмпирический докинг (AutoDock Vina), геометрические алгоритмы пространственного позиционирования и полуэмпирические квантовые методы расчета (xTB / CREST) для автоматического и стабильного предсказания 3D-структур.



## ✨ Основные возможности

* **Строгая изоляция запусков:** Каждый расчет выполняется в изолированной директории `run/<имя_проекта>/`. Временные файлы xTB, CREST и Vina пишутся только туда, что предотвращает конфликты при параллельных запусках.
* **Продвинутая геометрия по умолчанию:** Автоматическое совмещение центров масс Хозяина и Гостя с итеративным разнесением по координатной оси до полного исчезновения стерических наложений.
* **Интеллектуальный докинг через AutoDock Vina:** Включается опциональным флагом `--vina`. Автоматически строит бокс вокруг Хозяина и укладывает Гостя в наиболее энергетически выгодную полость с помощью генетического алгоритма.
* **Гибкое управление зарядом:** Поддерживает автоматический подсчет формального заряда на уровне графа RDKit либо принудительное ручное задание через флаг `--chrg` для корректного электронного описания системы в квантовом движке.
* **Конвейер xTB/CREST:** Сквозная оптимизация геометрии, конформационный поиск (CREST) и финальный расчет электронной структуры.
* **Контроль топологии:** Проверка сохранения химических связей после жесткой квантовой оптимизации (защита от нефизичного разрыва молекул).



## 📦 Подготовка окружения (Установка)

Для работы конвейера требуется менеджер пакетов **Conda** или **Mamba**. Все необходимые зависимости (включая xTB, CREST, RDKit и Vina) зафиксированы в файле `environment.yml`.

1. **Создайте виртуальное окружение из файла конфигурации:**
```bash
   conda env create -f environment.yml

```

2. **Активируйте созданное окружение:**

```bash
   conda activate supra_env

```



## 🚀 Параметры запуска (Интерфейс CLI)

Программа запускается через файл `main.py` и поддерживает следующие аргументы командной строки:

### 1. Источники данных (Ввод)

* `-host "SMILES или путь"` — Структура Хозяина. Поддерживаются строки SMILES, а также файлы `.sdf`, `.mol2`, `.pdb`.
* `-guest "SMILES или путь"` — Структура Гостя (опционально, используется для сборки комплексов).
* `-i "путь к файлу"` — Одиночный входной файл (например, уже готовый комплекс или изолированная молекула для оптимизации).
* `-o "имя"` — Пользовательское имя выходного проекта. По умолчанию генерируется автоматически на основе названий входных молекул.

### 2. Супрамолекулярные флаги

* `--vina` — Активировать предварительный эмпирический докинг через AutoDock Vina вместо дефолтного геометрического алгоритма.
* `--chrg <int>` — Вручную задать суммарный формальный заряд всей системы (крайне важно для заряженных комплексов и ионов). Если флаг опущен, заряд рассчитывается автоматически как сумма зарядов атомов RDKit.

### 3. Настройки квантового движка (xTB / CREST)

* `--gfn {0, 1, 2, ff}` — Уровень теории xTB (по умолчанию: `1` [GFN1-xTB]). Для огромных систем рекомендуется использовать силовое поле `ff`. Для нековалентных взаимодействий рекомендуется использовать уровень теории 1 или 2. 
* `--mode {standard, quick, mquick}` — Интенсивность конформационного поиска в CREST (рекомендуется mquick):


* `-t`, `--threads <int>` — Количество выделяемых процессорных потоков (по умолчанию: `6`).
* `--rebuild` — Принудительно пересобрать 3D-координаты молекул из графа RDKit, даже если на вход подан 3D-файл.
* `--noreftopo` — Отключить строгую проверку топологии связей RDKit после квантовой оптимизации.

## 🧠 Физико-химические особенности алгоритмов


### Взаимодействие локального фикса RDKit и флага --chrg

Локальная очистка зарядов (`fix_formal_charges`) работает исключительно с химическим графом (топологией) молекулы внутри RDKit до начала расчетов, чтобы молекула стала химически валидной, правильно прошла санитацию (`SanitizeMol`) и сгенерировала корректное число водородов (например, расставляет локальные плюсы на четвертичных азотах N+).

Флаг `--chrg` управляет глобальным электронным состоянием всей системы в xTB/CREST. Квантовые программы ничего не знают про «формальные заряды» RDKit — они принимают на вход только геометрию (XYZ-координаты) и общий заряд.

Передавая `--chrg`, вы принудительно указываете движку xTB точное количество электронов в системе. Квантовый движок рассчитывает честную волновую функцию для этой геометрии. В финальном SDF-файле координаты будут строго соответствовать квантовому расчету с вашим ручным зарядом, а локальные обозначения знаков (+/-) на атомах сохранятся для правильной химической нотации в визуализаторах.

## 🔬 Демонстрационный тест: Комплекс 18-Краун-6 и Метиламмония

Система «18-Краун-6 + Катион Метиламмония» — это классический пример супрамолекулярного включения «хозяин-гость». Стабилизация комплекса происходит за счет точного геометрического соответствия полости краун-эфира и треножника водородных связей группы -NH3+ катиона. Общий заряд системы равен +1.

### Вариант 1: Запуск по умолчанию (Продвинутая геометрия)

В этом режиме алгоритм совместит геометрические центры молекул и аккуратно разведет их на расстояние 2.2 Å, чтобы избежать пересечения Ван-дер-Ваальсовых сфер, после чего xTB стянет их за счет невалентных сил.

```bash
python main.py -host "C1COCCOCCOCCOCCOCCO1" -guest "C[NH3+]" -o crown_geometric --mode mquick

```

### Вариант 2: Запуск с использованием AutoDock Vina

Флаг `--vina` заставит программу рассчитать трехмерный бокс вокруг краун-эфира и с помощью генетического алгоритма инкапсулировать катион метиламмония точно в центр полости макроцикла перед началом квантовой оптимизации.

```bash
python main.py -host "C1COCCOCCOCCOCCOCCO1" -guest "C[NH3+]" --vina -o crown_vina --mode mquick

```

## 📁 Структура папок проекта

Конвейер поддерживает строгую изоляцию всех расчетных процессов внутри папки `run/`:

```text
.
├── main.py                  # Главный CLI интерфейс
├── environment.yml          # Конфигурация окружения Conda
├── .gitignore               # Файл скрытия мусора Git
├── supramol/                # Пакет с исходным кодом модулей
│   ├── __init__.py
│   ├── config.py            # Настройка среды и патч путей GFN0
│   ├── loader.py            # Загрузка, валидация и генерация 3D RDKit
│   ├── docking.py           # Геометрический алгоритм и AutoDock Vina
│   ├── engine.py            # Движок управления xTB / CREST
│   └── utils.py             # Вспомогательные утилиты парсинга XYZ
├── run/                     # ИГНОРИРУЕТСЯ GIT. Временные рабочие папки
│   └── crown_vina/          # Изолированная папка конкретного запуска
│       ├── host_rebuild_tmp.sdf
│       ├── guest_rebuild_tmp.sdf
│       ├── vina_docking/    # Входные/выходные PDBQT файлы AutoDock Vina
│       └── quantum_run/     # Логи, траектории и файлы расчетов xTB/CREST
└── result/                  # ИГНОРИРУЕТСЯ GIT. Готовые результаты
    └── crown_vina_final.sdf # Финальный квантово-оптимизированный 3D комплекс

```
